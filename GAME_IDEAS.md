## 🔥 개발 예정 (우선순위순)

### 1. 블록셈 (Block Sum) ⭐ 다음 개발 예정

> 일본 전통놀이 '다루마 오토시'에서 영감을 받은 덧셈 퍼즐

**모티브**: 말랑말랑 두뇌학원 - 덧셈 블록

**게임 규칙**:

1. 숫자가 쓰인 블록 4~6개가 탑처럼 세로로 쌓여 등장
2. 목표 숫자가 제시됨
3. 블록을 터치 후 **왼쪽 스와이프**로 제거
4. **남은 블록의 합 = 목표 숫자**가 되면 성공!
5. 블록을 적게 제거할수록 높은 점수
6. 목표 달성 불가능 상태 → 실패 처리
7. **60초** 제한시간 내 최대한 많이 클리어

**난이도 시스템**:
| 난이도 | 블록 개수 | 숫자 범위 | 승급 조건 |
|--------|----------|----------|----------|
| 하 (Easy) | 4개 | 1~9 | 3회 성공 시 → 중 |
| 중 (Normal) | 5개 | 1~12 | 3회 성공 시 → 상 |
| 상 (Hard) | 6개 | 1~15 | 유지 |

**기술 고려사항**:

- 스와이프 제스처 감지 (Phaser input)
- 블록 제거 시 위 블록들 자연스럽게 낙하 애니메이션
- "달성 가능 여부" 검증: 조합 수가 적으므로 (2^6 = 64) 병목 없음
- 물리 시뮬레이션 필요 없음 (단순 tween으로 처리)

**UI 참고** (말랑말랑 두뇌학원):

- 블록이 세로로 쌓임
- 목표 숫자가 오른쪽에 크게 표시
- 제한 시간 상단
- 점수 우상단

**예상 개발 시간**: 1~2일

### 2. 매스 플라이트 (Math Flight)

> 종스크롤 러닝 액션을 차용한 계산 두뇌 게임

**모티브**: 드래곤 플라이트 + 수학 퍼즐

**게임 규칙**:

1. 종스크롤 화면, 하단에 캐릭터 배치 (상하 고정, 좌우 드래그 이동)
2. 캐릭터는 **'파워(숫자)'**를 보유
3. 5줄의 운석이 위에서 떨어짐 (각 운석에 숫자 표시)
4. **충돌 판정**:
   - 운석 숫자 > 내 파워 → 💀 라이프 손실
   - 운석 숫자 < 내 파워 → ✅ 운석 숫자가 파워에 더해짐
5. **핵심 전략**: 파워보다 낮은 수 중 **가장 큰 수**를 선택해야 최적!

**난이도 밸런스 알고리즘** (핵심):

| 상황           | 다음 운석 구성                         |
| -------------- | -------------------------------------- |
| 정답 선택 시   | 실패 2개 + 성공 3개 (적정 난이도 유지) |
| 차선책 선택 시 | 실패 운석 비율 증가 → 난이도 상승      |
| 연속 정답 시   | 점진적 정상화                          |

**마이너스 운석 턴**:

- 일정 주기로 등장 (파워 과다 증가 방지)
- 모든 운석이 **음수** → 파워 강제 감소
- 유저는 **절대값이 가장 작은 운석** 선택해야 손해 최소화

**기술 고려사항**:

- 터치 드래그 → 캐릭터 좌우 이동
- 운석 생성 알고리즘 (밸런스가 게임의 핵심)
- 5레인 시스템 (좌-중좌-중앙-중우-우)
- 충돌 감지 + 파워 계산
- 무한 러닝 (제한시간 or 라이프 소진까지)

**UI 구성**:

```
┌─────────────────────────────────┐
│  ❤️❤️❤️        파워: 42        │  ← HUD
├─────────────────────────────────┤
│                                 │
│   [15]  [38]  [67]  [23]  [41] │  ← 떨어지는 운석들
│                                 │
│         [52]  [8]   [33]       │
│                                 │
│              [🚀]              │  ← 캐릭터 (좌우 이동)
└─────────────────────────────────┘
```

**예상 개발 시간**: 2~3일 (밸런스 조정 포함)

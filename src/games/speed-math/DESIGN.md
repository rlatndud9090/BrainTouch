# Speed Math (스피드 계산) - 게임 설계 문서

> 닌텐도 DS '매일매일 두뇌 트레이닝'의 계산 게임에서 영감을 받은 사칙연산 스피드 퀴즈

---

## 📌 게임 개요

| 항목            | 내용                                |
| --------------- | ----------------------------------- |
| **게임명**      | Speed Math (스피드 계산)            |
| **게임 ID**     | `speed-math`                        |
| **장르**        | 두뇌 트레이닝 / 퍼즐                |
| **목표**        | 20문제를 최대한 빠르게 풀기         |
| **핵심 메트릭** | 클리어 시간 (초 단위, 소수점 2자리) |

## 🎮 게임 규칙

### 기본 규칙

1. 총 **20문제**의 사칙연산 문제 출제
2. 문제를 풀기 시작하면 **타이머 시작**
3. 20문제를 모두 맞추면 **타이머 정지**
4. **최종 기록 = 걸린 시간** (낮을수록 좋음)
5. **확인 버튼 없음** - 입력 즉시 정답 체크
   - 한자리 정답: 한자리 입력 시 즉시 통과
   - 두자리 정답: 두자리 입력 시 즉시 통과
   - 오답: 두자리 입력 완료 시에만 X표시 → 자동 지움 → 재입력
6. 전체 지우기 [C] 버튼만 제공 (스피디한 재입력)

### 연산 난이도

| 연산           | 첫번째 숫자 | 두번째 숫자 | 결과 범위 | 예시       |
| -------------- | ----------- | ----------- | --------- | ---------- |
| **더하기 (+)** | 1~9         | 1~9         | 2~18      | 3 + 7 = 10 |
| **빼기 (-)**   | 1~19        | 1~9 (≤OP1)  | 0~18      | 15 - 8 = 7 |
| **곱하기 (×)** | 1~9         | 1~9         | 1~81      | 6 × 7 = 42 |
| **나누기 (÷)** | ❌ 없음     | -           | -         | -          |

### 빼기 특별 규칙

- **음수 결과 없음**: 첫번째 숫자 ≥ 두번째 숫자로 제한
- 문제 생성 시 검증 로직:
  ```typescript
  // 빼기 문제 생성 (음수 방지)
  const op1 = randomInt(1, 19);
  const op2 = randomInt(1, Math.min(9, op1)); // op1 이하로 제한
  ```
- ⚠️ 검증이 병목이 되지 않도록 생성 단계에서 범위 제한

## 🎯 입력 방식

### 모드 1: 필기 입력 (Handwriting) *(내부 검토 중, 사용자 미노출)*

```
┌─────────────────────────────────┐
│  3/20                   12.45초 │  ← 진행률(좌상단), 타이머(우상단, 작게)
├─────────────────────────────────┤
│                                 │
│         7 + 8 = [15]            │  ← 문제 + 입력값이 ? 자리에 표시
│                                 │
├─────────────────────────────────┤
│                                 │
│    ┌───────────────────┐        │
│    │                   │        │  ← 필기 캔버스
│    │    (그리는 영역)   │        │     (숫자를 손으로 그림)
│    │                   │        │
│    └───────────────────┘        │
│                                 │
│           [지우기]              │  ← 지우기 버튼만
└─────────────────────────────────┘
```

**입력 흐름:**

1. 숫자를 그리면 실시간으로 인식 → 문제의 `?` 자리에 표시
2. 인식 결과가 정답과 일치하면 → 초록색 플래시 + 즉시 다음 문제
3. 두자리 인식 후 오답이면 → 빨간색 흔들림 + 자동 지움 + 재입력 대기

**기술 스택:**

- **TensorFlow.js** - 머신러닝 추론 엔진
- **MNIST 모델** - 손글씨 숫자 인식 (0-9)
- **Canvas API** - 필기 입력 캡처

**구현 고려사항:**

1. 두 자리 숫자 인식 (10~81)
   - 캔버스를 좌/우로 분할하여 각각 인식
   - 또는 연속 필기 후 세그멘테이션
2. 모델 사전 로딩 (게임 시작 전)
3. 실시간 피드백 (그리는 중 인식 결과 표시)

**성능 목표:**

- 추론 시간: < 50ms (모바일 웹뷰 기준)
- 인식 정확도: > 95%

### 모드 2: 숫자패드 입력 (Keypad)

```
┌─────────────────────────────────┐
│  3/20                   12.45초 │  ← HUD (진행률, 타이머)
├─────────────────────────────────┤
│                                 │
│     2 × 3 = [6_]                │  ← 현재 문제 (크게, 강조)
│     ─────────────               │     입력값이 옆에 표시
│     7 - 3 =                     │  ← 다음 문제 (미리보기)
│                                 │
│     6 × 6 =                     │  ← 그다음 문제 (흐리게)
│                                 │
├─────────────────────────────────┤
│                                 │
│      [1]   [2]   [3]            │
│      [4]   [5]   [6]            │  ← 숫자 패드
│      [7]   [8]   [9]            │
│            [0]   [C]            │
│                                 │
└─────────────────────────────────┘
```

**3문제 동시 표시의 장점:**

- 현재 문제를 풀면서 **다음 문제를 미리 계산** 가능
- 원작 '매일매일 두뇌 트레이닝' 방식 재현
- 더 빠른 플레이 가능

**입력 흐름:**

1. 숫자 입력 → 현재 문제의 `?` 자리에 실시간 표시
2. 한자리 정답: 첫 숫자 입력 시 정답이면 즉시 통과
3. 두자리 정답: 두번째 숫자 입력 시 정답이면 즉시 통과
4. 오답 판정: 두자리 입력 완료 후에만 체크
5. 오답 시: 빨간색 흔들림 → 자동 지움 → 재입력 대기
6. 정답 시: 문제가 위로 스크롤 + 새 문제가 아래서 등장

**일반 모드:**

- 숫자 배치: 1-9 고정, 계산기 레이아웃

**어려움 모드 (랜덤 패드):**

- 매 문제마다 숫자 위치 랜덤 셔플
- 시각적 혼란으로 난이도 상승

```
[어려움 모드 예시]
      [5]   [2]   [9]
      [1]   [7]   [4]
      [3]   [6]   [8]
            [0]   [C]
```

```

```


- **진행률 (좌상단)**: `3/20` 형식, 눈에 잘 띄게
- **타이머 (우상단)**: 소수점 2자리, 작고 은은하게 (집중 방해 최소화)
  - 색상: 연한 회색 (#a0a0a0)
  - 폰트 크기: 진행률보다 작게

```
         7 + 8 = [__]     ← 입력 전 (빈 박스)
         7 + 8 = [1_]     ← 한자리 입력 중
         7 + 8 = [15]     ← 두자리 입력 완료
```

- 입력값이 `?` 자리에 직접 채워짐
- 정답 시: 초록색 플래시 효과
- 오답 시: 빨간색 + 흔들림 효과

## 📊 데이터 구조

### 문제 (Question)

```typescript
interface Question {
  id: number; // 문제 번호 (1-20)
  operand1: number; // 첫번째 숫자
  operand2: number; // 두번째 숫자
  operator: '+' | '-' | '×'; // 연산자
  answer: number; // 정답
  displayText: string; // 화면 표시용 "7 + 8 = ?"
}
```

### 게임 상태 (GameState)

```typescript
interface GameState {
  mode: 'handwriting' | 'keypad-normal' | 'keypad-hard';
  currentQuestion: number; // 현재 문제 인덱스 (0-19)
  questions: Question[]; // 20개 문제 배열
  startTime: number; // 게임 시작 타임스탬프
  elapsedTime: number; // 경과 시간 (ms)
  isPlaying: boolean;
  wrongAttempts: number; // 현재 문제 오답 횟수
}
```

### 게임 결과 (GameResult)

```typescript
interface GameResult {
  mode: string;
  totalTime: number; // 총 소요 시간 (ms)
  totalWrongAttempts: number; // 총 오답 횟수
  date: Date;
  isNewRecord: boolean;
}
```

## ⚡ 성능 최적화

### TensorFlow.js 최적화

1. **모델 사전 로딩**: PreloadScene에서 로딩
2. **모델 캐싱**: IndexedDB에 저장
3. **WebGL 백엔드 우선**: GPU 가속 활용
4. **텐서 메모리 관리**: 사용 후 즉시 dispose()

### 입력 최적화

1. **디바운싱**: 필기 인식 요청 throttle (100ms)
2. **캔버스 최적화**: 28x28 리사이즈 후 추론
3. **패드 터치**: 즉각적인 피드백 (< 16ms)

## 🎮 게임 규칙

### 기본 규칙

1. 총 **20문제**의 사칙연산 문제 출제
2. 문제를 풀기 시작하면 **타이머 시작**
3. 20문제를 모두 맞추면 **타이머 정지**
4. **최종 기록 = 걸린 시간** (낮을수록 좋음)
5. **확인 버튼 없음** - 입력 즉시 정답 체크
   - 한자리 정답: 한자리 입력 시 즉시 통과
   - 두자리 정답: 두자리 입력 시 즉시 통과
   - 오답: 두자리 입력 완료 시에만 X표시 → 자동 지움 → 재입력
6. 전체 지우기 [C] 버튼만 제공 (스피디한 재입력)

### 연산 난이도

| 연산           | 첫번째 숫자 | 두번째 숫자 | 결과 범위 | 예시       |
| -------------- | ----------- | ----------- | --------- | ---------- |
| **더하기 (+)** | 1~9         | 1~9         | 2~18      | 3 + 7 = 10 |
| **빼기 (-)**   | 1~19        | 1~9 (≤OP1)  | 0~18      | 15 - 8 = 7 |
| **곱하기 (×)** | 1~9         | 1~9         | 1~81      | 6 × 7 = 42 |
| **나누기 (÷)** | ❌ 없음     | -           | -         | -          |

### 빼기 특별 규칙

- **음수 결과 없음**: 첫번째 숫자 ≥ 두번째 숫자로 제한
- 문제 생성 시 검증 로직:
  ```typescript
  // 빼기 문제 생성 (음수 방지)
  const op1 = randomInt(1, 19);
  const op2 = randomInt(1, Math.min(9, op1)); // op1 이하로 제한
  ```
- ⚠️ 검증이 병목이 되지 않도록 생성 단계에서 범위 제한

## 🎯 입력 방식

### 숫자패드 입력 (Keypad)

```
┌─────────────────────────────────┐
│  3/20                   12.45초 │  ← HUD (진행률, 타이머)
├─────────────────────────────────┤
│                                 │
│     2 × 3 = [6_]                │  ← 현재 문제 (크게, 강조)
│     ─────────────               │     입력값이 옆에 표시
│     7 - 3 =                     │  ← 다음 문제 (미리보기)
│                                 │
│     6 × 6 =                     │  ← 그다음 문제 (흐리게)
│                                 │
├─────────────────────────────────┤
│                                 │
│      [1]   [2]   [3]            │
│      [4]   [5]   [6]            │  ← 숫자 패드
│      [7]   [8]   [9]            │
│            [0]   [C]            │
│                                 │
└─────────────────────────────────┘
```

**3문제 동시 표시의 장점:**

- 현재 문제를 풀면서 **다음 문제를 미리 계산** 가능
- 원작 '매일매일 두뇌 트레이닝' 방식 재현
- 더 빠른 플레이 가능

**입력 흐름:**

1. 숫자 입력 → 현재 문제의 `?` 자리에 실시간 표시
2. 한자리 정답: 첫 숫자 입력 시 정답이면 즉시 통과
3. 두자리 정답: 두번째 숫자 입력 시 정답이면 즉시 통과
4. 오답 판정: 두자리 입력 완료 후에만 체크
5. 오답 시: 빨간색 흔들림 → 자동 지움 → 재입력 대기
6. 정답 시: 문제가 위로 스크롤 + 새 문제가 아래서 등장

현재 숫자패드는 고정 레이아웃 1종만 제공합니다.
- 숫자 배치: `1-9`, 마지막 줄 `0`/`C`
- 입력 모드 전환 UI 없음
- 랜덤 패드와 필기 입력은 현재 구현에서 제거됨

## 🎨 UI/UX 설계

### 색상 팔레트

```css
--bg-primary: #1a1a2e      /* 메인 배경 (어두운 네이비) */
--bg-secondary: #16213e    /* 보조 배경 */
--accent-red: #e94560      /* 강조색 (틀림/중요) */
--accent-green: #4ecca3    /* 정답/성공 */
--accent-yellow: #ffc947   /* 경고/시간 */
--text-primary: #ffffff    /* 메인 텍스트 */
--text-secondary: #a0a0a0  /* 보조 텍스트 */
```

### 문제 표시 애니메이션

1. 새 문제: 페이드인 + 약간 위에서 슬라이드
2. 정답: 초록색 플래시 + 체크 아이콘
3. 오답: 빨간색 흔들림 + X 아이콘

### 상단 HUD 레이아웃

```
┌─────────────────────────────────┐
│  3/20                   12.45초 │
│  (진행률)              (타이머) │
└─────────────────────────────────┘
```

- **진행률 (좌상단)**: `3/20` 형식, 눈에 잘 띄게
- **타이머 (우상단)**: 소수점 2자리, 작고 은은하게 (집중 방해 최소화)
  - 색상: 연한 회색 (#a0a0a0)
  - 폰트 크기: 진행률보다 작게

### 문제 + 답 입력 영역

```
         7 + 8 = [__]     ← 입력 전 (빈 박스)
         7 + 8 = [1_]     ← 한자리 입력 중
         7 + 8 = [15]     ← 두자리 입력 완료
```

- 입력값이 `?` 자리에 직접 채워짐
- 정답 시: 초록색 플래시 효과
- 오답 시: 빨간색 + 흔들림 효과

## 📊 데이터 구조

### 문제 (Question)

```typescript
interface Question {
  id: number; // 문제 번호 (1-20)
  operand1: number; // 첫번째 숫자
  operand2: number; // 두번째 숫자
  operator: '+' | '-' | '×'; // 연산자
  answer: number; // 정답
  displayText: string; // 화면 표시용 "7 + 8 = ?"
}
```

### 게임 상태 (GameState)

```typescript
interface GameState {
  currentQuestionIndex: number; // 현재 문제 인덱스 (0-19)
  questions: Question[]; // 20개 문제 배열
  currentInput: string; // 현재 입력 중인 답
  startTime: number; // 게임 시작 타임스탬프
  elapsedTime: number; // 경과 시간 (ms)
  isPlaying: boolean;
}
```

### 게임 결과 (GameResult)

```typescript
interface GameResult {
  totalTime: number; // 총 소요 시간 (ms)
}
```

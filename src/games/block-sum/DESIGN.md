| 항목          | 내용                             |

## 🎮 게임 규칙

### 기본 규칙

1. 숫자가 쓰인 블록 4~6개가 탑처럼 세로로 쌓여 등장
2. 화면에 **목표 숫자**가 제시됨
3. 블록을 터치한 뒤 **좌/우로 빠르게 스와이프**하면 블록 제거
4. 제거 후 **남은 블록들의 합 = 목표 숫자**가 되면 즉시 성공 처리
5. 성공하면 점수 획득, 다음 라운드 진행
6. 라운드마다 제한시간 안에 풀지 못하면 실패(라이프 감소)

### 실패 조건

- 블록을 너무 많이 제거해서 **목표 달성이 불가능**해지면 자동 실패
- 실패 시 다음 라운드로 넘어감 (점수 없음)

### 점수 시스템

```
점수 = 100 × 남은 블록 수 × 점수 배수

점수 배수:
- `difficultyScore` 기반으로 `1.00 ~ 3.00`
- 계산식: `1 + difficultyScore * 2`
- 블록 개수 축은 라운드 구간에 따라 자동 반영되고,
  나머지 축(`numberRange`, `targetComplexity`)은 성공 시 점진적으로 상승

예시: `difficultyScore = 0.65`에서 4개 블록 남기고 성공
      → 100 × 4 × 2.30 = 920점
```

## 📊 난이도 시스템

현재 구현은 `utils/DifficultyDirector.ts`를 기준으로 세 축을 조합합니다.

### 1. 블록 개수 축 (`blockCount`)

- 라운드 진행에 따라 자동 적용
- `4개 → 5개 → 6개`

### 2. 숫자 범위 축 (`numberRange`)

| 레벨 | 숫자 범위 |
| ---- | --------- |
| 0    | 1~5       |
| 1    | 1~7       |
| 2    | 1~9       |
| 3    | 1~12      |

### 3. 목표 복잡도 축 (`targetComplexity`)

| 레벨 | 최대 제거 수 | 선호 제거 수 | 목표 숫자 최대 |
| ---- | ------------ | ------------ | -------------- |
| 0    | 1            | 1            | 15             |
| 1    | 2            | 1            | 22             |
| 2    | 2            | 2            | 30             |
| 3    | 3            | 2            | 제한 없음      |

### 시간 레벨

- 라운드 구간에 따라 `6초 → 5초 → 4초`
- 시간 레벨은 `1~24라운드: 6초`, `25~48라운드: 5초`, `49라운드+: 4초`로 24라운드 간격 상승
- 블록 개수 축은 매 12라운드마다 오르는 것이 아니라 `1~12라운드: 4개`, `13~36라운드: 5개`, `37라운드+: 6개`로 13라운드와 37라운드에서만 증가

### 승급/하향 규칙

- 성공 시 `numberRange` 또는 `targetComplexity` 축이 가중 랜덤으로 상승
- 한 번에 너무 큰 난이도 점프가 생기면 상승이 보류될 수 있음
- 실패 시에는 `targetComplexity`를 먼저, 그 다음 `numberRange`를 1단계 하향
- `blockCount` 축은 실패로 직접 하향되지 않음

### 핵심 알고리즘

#### 목표 숫자 생성

1. 난이도에 맞는 블록 생성 (랜덤 숫자)
2. 모든 부분집합의 합 계산 (비트마스크, 최대 2^6 = 64개)
3. 가능한 합 중 랜덤 선택 → 목표 숫자로 설정

```typescript
// 모든 가능한 합 계산
for (let mask = 0; mask < 1 << n; mask++) {
  // mask 비트가 1인 블록은 제거, 0인 블록은 유지
  // 유지된 블록들의 합 계산
}
```

#### 달성 가능 여부 체크

- 블록 제거 시마다 남은 블록으로 목표 달성 가능한지 검사
- 불가능하면 즉시 실패 처리

### 스와이프 감지

```typescript
// pointerdown에서 시작 위치 저장
// pointerup에서 이동 거리 계산
const dx = pointer.x - startX;
if (Math.abs(dx) >= 30 && speed >= 0.8 && Math.abs(dy) < Math.abs(dx)) {
  // 좌/우 빠른 스와이프 → 블록 제거
}
```

### 낙하 애니메이션

- 블록 제거 시 Tween으로 부드러운 낙하
- `Bounce.easeOut` 이징으로 자연스러운 착지

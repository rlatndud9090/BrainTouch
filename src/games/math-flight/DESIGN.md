## 🎮 게임 규칙

### 핵심 컨셉: "중간값 찾기"

- 3개의 운석이 각각 다른 숫자를 가지고 동시에 떨어짐
- **정확히 중간값** 운석에 부딪히면 → ✅ 점수 획득
- **가장 큰 수** / **가장 작은 수** 운석에 부딪히면 → ❌ 라이프 손실

### 예시

```
운석 3개: [14, 22, 31]
           ❌  ✅  ❌
          실패 정답 실패
```

### 점수 시스템

| 행동                   | 점수            |
| ---------------------- | --------------- |
| **중간값** 정확히 충돌 | +200점          |
| 실패 (최소/최대 운석)  | 라이프 -1       |
| 4회 성공               | 난이도 축 1단계 상승 |

### 라이프 시스템

- 초기 라이프: 3개
- 라이프 0 → 게임 오버

## 🎯 난이도 밸런스 알고리즘

### 숫자 범위: **0 ~ 50**

### 난이도 축

| 축              | 설명                           | 레벨 예시 |
| --------------- | ------------------------------ | --------- |
| 낙하 속도       | 운석이 내려오는 속도            | 1.00x → 1.70x |
| 숫자 간격 범위  | 중간값과 양옆 숫자의 차이 폭     | 9~14 → 3~5 |

### 증감 규칙

- 성공 4회마다 축 1개를 랜덤 상승
- 동일 축 3연속 상승 금지
- 실패 시 난이도 1단계 하향 (우선순위: 숫자 간격 → 낙하 속도)
- 모든 축이 최대 레벨이면 최고 난이도 유지

### 운석 생성 알고리즘

```typescript
generateMeteors(config: { minGap: number; maxGap: number }): MeteorData[] {
  // 1. gap 2개를 난이도 축 범위에서 샘플링
  // 2. 유효한 median 범위에서 중간값 생성
  // 3. [min, median, max] 3개 운석 생성
  // 4. 3개 레인(좌/중/우)에 셔플 배치
}
```

### 난이도 예시

```
초반: [10, 24, 38]
      간격 넓음 + 속도 완만 → 판단 쉬움

중반: [18, 24, 31]
      간격 좁아짐 → 중간값 인식 난이도 상승

후반: [22, 25, 28]
      좁은 간격 + 고속 낙하 → 빠른 판단/조작 필요
```

### 색상 팔레트


### 애니메이션

1. **운석 낙하**: 위에서 아래로 (속도 점진적 증가)
2. **충돌 성공**: 노란색 플래시 + 점수 팝업
3. **충돌 실패**: 빨간색 플래시 + 화면 흔들림
4. **난이도 변화**: 내부 축 레벨만 갱신 (별도 텍스트 없음)

## ⚡ 기술 고려사항

### 터치 조작

```typescript
// 드래곤 플라이트 스타일 자유 이동
// 터치 down → 즉시 해당 X 좌표로 이동
// 드래그 → 실시간 따라 이동
this.input.on('pointerdown', (pointer) => {
  this.movePlayerToX(pointer.x);
});
this.input.on('pointermove', (pointer) => {
  if (pointer.primaryDown) {
    this.movePlayerToX(pointer.x);
  }
});
```

### 운석 생성 주기

- **1턴 = 3개 운석 동시 낙하**
- 충돌 후 다음 웨이브 생성
- 속도: 축 레벨에 따라 단계적으로 증가 (1.00x → 1.70x)

### 충돌 감지

```typescript
// 판정 라인 도달 시 플레이어 레인과 운석 레인 비교
if (meteor.data.lane === playerLane) {
  handleCollision(meteor);
}
```

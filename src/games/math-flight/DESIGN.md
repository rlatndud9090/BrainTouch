## 🎮 게임 규칙

### 핵심 컨셉: "중간값 찾기"

- 5개의 운석이 각각 다른 숫자를 가지고 떨어짐
- **가장 큰 수** / **가장 작은 수** 운석에 부딪히면 → ❌ 라이프 손실
- **중간 3개** 운석에 부딪히면 → ✅ 점수 획득
- **정확히 중간값** 운석에 부딪히면 → ⭐ **2배 점수**

### 예시

```
운석 5개: [12, 18, 25, 32, 38]
           ❌  ✅  ⭐  ✅  ❌
          실패 성공 2배! 성공 실패
```

### 점수 시스템

| 행동                   | 점수              |
| ---------------------- | ----------------- |
| 성공 운석 충돌         | 기본 점수 (100점) |
| **중간값** 정확히 충돌 | **2배 점수**      |
| 실패 (최소/최대 운석)  | 라이프 -1         |

### 라이프 시스템

- 초기 라이프: 3개
- 라이프 0 → 게임 오버

## 🎯 난이도 밸런스 알고리즘

### 숫자 범위: **0 ~ 50**

### 난이도별 분포

| 난이도 | 진행 턴 | 성공 범위 (±) | 실패 범위 (±) | 속도 배수 |
| ------ | ------- | ------------- | ------------- | --------- |
| 초반   | 1~10    | 5~8           | 10~15         | 1.0x      |
| 중반   | 11~25   | 4~7           | 7~11          | 1.3x      |
| 후반   | 26+     | 3~5           | 4~8           | 1.6x      |

### 운석 생성 알고리즘

```typescript
generateMeteors(midValue: number, difficulty: string): number[] {
  // 1. 중간값 결정 (10~40 범위, 경계 여유)
  // 2. 성공 범위에서 2개 숫자 생성
  // 3. 실패 범위에서 2개 숫자 생성
  // 4. 중간값 포함하여 총 5개
  // 5. 셔플하여 5개 레인에 배치

  // 결과: [실패, 성공, 중간값, 성공, 실패] (정렬 시)
}
```

### 난이도 예시

```
초반 (턴 5): [8, 19, 25, 31, 42]
             간격 넓음 → 구분 쉬움

중반 (턴 15): [15, 19, 25, 31, 35]
              간격 좁아짐 → 판단 빨라야 함

후반 (턴 30): [18, 21, 25, 29, 32]
              거의 연속 → 빠른 두뇌 회전 필요!
```

## ⚡ 기술 고려사항

### 터치 조작

```typescript
// 드래곤 플라이트 스타일 자유 이동
// 터치 down → 즉시 해당 X 좌표로 이동
// 드래그 → 실시간 따라 이동
this.input.on('pointerdown', (pointer) => {
  this.movePlayerToX(pointer.x);
});
this.input.on('pointermove', (pointer) => {
  if (pointer.primaryDown) {
    this.movePlayerToX(pointer.x);
  }
});
```

### 운석 생성 주기

- **1턴 = 5개 운석 동시 낙하**
- 충돌 후 다음 웨이브 생성
- 속도: 난이도에 따라 증가 (3초 → 2.3초 → 1.9초)

### 충돌 감지

```typescript
// 거리 기반 충돌 (반경 35px)
const distance = Math.sqrt(dx * dx + dy * dy);
if (distance < 35) {
  handleCollision(meteor);
}
```

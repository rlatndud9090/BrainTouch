## 🎮 게임 규칙

### 기본 규칙

1. 종스크롤 화면, 캐릭터는 하단 고정 (좌우 이동만 가능)
2. 캐릭터는 **'파워(숫자)'**를 보유 (초기값: 10)
3. 5개 레인에서 운석이 위에서 아래로 떨어짐
4. 운석과 충돌 시:
   - **운석 숫자 > 내 파워** → ❌ 라이프 1개 손실
   - **운석 숫자 ≤ 내 파워** → ✅ 운석 숫자가 파워에 더해짐 + 점수 획득
5. 라이프 0개 → 게임 오버

### 점수 시스템

| 행동                          | 점수             |
| ----------------------------- | ---------------- |
| 성공 운석 흡수                | 흡수한 숫자 × 10 |
| 최적 선택 (가장 큰 성공 운석) | 추가 보너스 ×1.5 |
| 마이너스 턴 최적 선택         | 추가 보너스 ×2   |

### 핵심 전략

> **파워보다 낮은 수 중 가장 큰 수**를 선택해야 최적!

- 너무 작은 운석만 흡수 → 파워 증가 느림 → 다음 턴 선택지 줄어듦
- 큰 운석 욕심내다 실패 → 라이프 손실

## 🎯 난이도 밸런스 알고리즘

### 운석 생성 규칙

```typescript
// 현재 파워 기준으로 5개 운석 생성
generateMeteors(power: number): number[] {
  // 성공 가능 운석 3개 (power 이하)
  // 실패 운석 2개 (power 초과)
  // 그 중 하나는 "정답" (power 이하 중 최대값)
}
```

### 상황별 난이도 조정

| 상황          | 다음 운석 구성                 |
| ------------- | ------------------------------ |
| 정답 선택     | 성공 3개 + 실패 2개 (기본)     |
| 차선책 선택   | 성공 2개 + 실패 3개 (난이도 ↑) |
| 연속 정답 3회 | 성공 3개 + 실패 2개로 복귀     |
| 연속 실패 2회 | 성공 4개 + 실패 1개 (구제)     |

### 마이너스 운석 턴

- **발동 조건**: 파워가 50 이상일 때, 5턴마다 1회
- **특징**: 모든 운석이 음수
- **전략**: 절대값이 가장 작은 운석 선택 (손해 최소화)

```typescript
// 마이너스 턴 예시
// 파워: 67
// 운석: [-12, -8, -25, -3, -17]
// 정답: -3 (절대값 최소)
```

### 파워 상한선

- 최대 파워: 99
- 99 도달 시: 마이너스 턴 강제 발동

```

    │
    ▼
[GameScene] ─────── 게임 플레이
    │               - 캐릭터 좌우 이동
    │               - 운석 생성 & 낙하
    │               - 충돌 감지
    │               - 파워/점수 계산
    │               - 라이프 관리
    │

    │               - 총 점수
    │               - 생존 시간
    │               - 최고 파워
    ▼
[홈 or 재시작]
```

## 📊 데이터 구조

### 운석 (Meteor)

```typescript
interface Meteor {
  id: number;
  value: number; // 운석 숫자 (양수 또는 음수)
  lane: number; // 레인 (0-4)
  y: number; // Y 좌표
  isOptimal: boolean; // 현재 턴의 최적 선택인지
  sprite: Phaser.GameObjects.Container;
}
```

### 게임 상태 (GameState)

```typescript
interface GameState {
  power: number; // 현재 파워 (초기값: 10)
  score: number; // 현재 점수
  lives: number; // 남은 라이프 (초기값: 3)
  currentLane: number; // 캐릭터 현재 레인 (0-4)
  turnCount: number; // 총 턴 수
  consecutiveOptimal: number; // 연속 정답 횟수
  isMinusTurn: boolean; // 마이너스 턴 여부
  startTime: number; // 게임 시작 시간
  maxPower: number; // 최고 파워 기록
}
```

### 게임 결과 (GameResult)

```typescript
interface GameResult {
  totalScore: number;
  survivalTime: number; // 생존 시간 (ms)
  maxPower: number; // 도달한 최고 파워
  turnCount: number; // 총 진행 턴
}
```

## ⚡ 기술 고려사항

### 터치 조작

```typescript
// 드래그로 캐릭터 좌우 이동
this.input.on('pointermove', (pointer) => {
  if (pointer.isDown) {
    // 터치 X 좌표 → 가장 가까운 레인으로 이동
    const targetLane = this.getLaneFromX(pointer.x);
    this.movePlayerToLane(targetLane);
  }
});
```

### 운석 생성 주기

- **1턴 = 5개 운석 동시 낙하**
- 운석 낙하 속도: 3초 (상단 → 하단)
- 다음 턴 시작: 현재 운석들이 화면 50% 지점 도달 시

### 충돌 감지

```typescript
// 운석이 캐릭터 Y 좌표에 도달했을 때
// 같은 레인이면 충돌 처리
if (meteor.lane === playerLane && meteor.y >= playerY) {
  handleCollision(meteor);
}
```
